# Meta-Review Skill

> **Handled DIRECTLY by the orchestrator.** Do not delegate to a subagent.

## Start

Announce: "Running meta-review of kiro-config effectiveness."

---

## Phase 1: Automated Findings

Read `~/.kiro/knowledge/improvement-suggestions.md` (generated by self-improve hook).
If it exists and has findings, list them first — these are mechanical issues to fix first.

## Phase 2: Skill Coverage

For each skill in `skills/`:
- Does the trigger description match how users actually phrase requests?
  Cross-check against `knowledge/episodes.md` for real phrasing patterns.
- Are any triggers so specific they likely never fire?
- Are there recurring patterns in `episodes.md` with no matching skill?

## Phase 3: Steering Effectiveness

For each steering doc in `.kiro/steering/`:
- Is the content accurate for the current project structure?
- Is there duplication with `docs/context/` (if it exists)?
- Are any rules too vague to be actionable?

## Phase 4: Knowledge Hygiene

- Episodes that appear 3+ times → candidate for promotion to a rule
- Rules that haven't been relevant in recent episodes → candidate for removal
- Gotchas that are now covered by config → remove to reduce noise

## Phase 5: Routing Review

Review the orchestrator's routing config:
- Are routing triggers well-separated with no ambiguous overlaps?
- Does the "handle directly" list cover all planning/conversation patterns?
- Are there agent capabilities that are never routed to?

---

## Output Format

```
## Meta-Review — YYYY-MM-DD

### Automated Findings (from self-improve hook)
- [findings or 'none']

### Skill Gaps
- [missing or misaligned skills]

### Steering Updates
- [file] — [what to change and why]

### Knowledge Hygiene
- [promotions, removals, updates]

### Routing Improvements
- [specific trigger changes]

### No Action Needed
[items reviewed and found current]
```

Present report to user for approval. **Never auto-apply changes.**
